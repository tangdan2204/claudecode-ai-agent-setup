# 反复出现问题追踪表（自我学习引擎）

> 本文件是「举一反三」机制的核心数据源。
> 每次修复 bug 或解决问题时，检查此表是否有同类记录，有则升级计数并触发对应层级。

## 升级链触发规则

| 出现次数 | 触发动作 | 归属阶段 |
|----------|----------|----------|
| 1次 | 修复即审计：同文件/同模块搜索同类问题 | 反思 |
| ≥2次 | 全局审计：Grep 全项目扫描 + 建立防御规则 | 进化 |
| ≥3次 | 深度反省：根因链分析 + 规则加固 + 提议创建自动化检测(hook/lint/test) | 进化 |
| ≥4次 | 举一反三：类比推断 + 跨项目迁移 + 预防性建议 | 进化 |

> **操作规范**: 更新计数后，立即检查是否跨越触发阈值。跨越则在当前任务的进化阶段执行对应动作。

## 模式记录格式

```
### [模式ID] 模式简称
- 类别: bug/配置/架构/性能/安全/逻辑
- 出现次数: N
- 首次发现: YYYY-MM-DD | 项目/文件
- 最近发现: YYYY-MM-DD | 项目/文件
- 根因: 一句话描述根本原因
- 修复模式: 标准修复方法
- 审计正则/Grep: 用于全局扫描的搜索模式
- 已审计项目: [项目列表]
- 已触发升级: [L1修复审计/L2全局审计/L3深度反省/L4举一反三]
- 状态: 活跃/已根治/已归档
```
